## 🐞 Fehler melden

1. Gehe zur Registerkarte **Issues**
2. Wähle die Vorlage „Bug report“
3. Gib an:
   - Schritte zur Reproduktion
   - Erwartetes vs. tatsächliches Verhalten
   - App- und Systemversion

---

## 📦 Pull Requests

1. Forke das Repository
2. Erstelle einen neuen Branch (`feature/dein-feature-name`)
3. Nimm deine Änderungen vor
4. Stelle sicher, dass alle Tests bestehen
5. Reiche einen Pull Request mit Beschreibung ein

---

## 🎨 Code-Stil

- Kotlin/Java: Android Studio-Konventionen
- XML: saubere Einrückung, Kommentare bei komplexen Layouts
- Markdown: UTF-8, H1–H3 Überschriften, Emojis optional
