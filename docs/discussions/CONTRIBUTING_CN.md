## 🐞 报告错误

1. 进入 **Issues** 标签页
2. 选择 “Bug report” 模板
3. 包含以下内容：
   - 重现步骤
   - 预期与实际行为
   - 应用和系统版本

---

## 📦 Pull Request 提交

1. Fork 仓库
2. 创建新分支 (`feature/功能名称`)
3. 修改代码
4. 确保测试通过
5. 提交 Pull Request 并附上说明

---

## 🎨 代码风格

- Kotlin/Java：遵循 Android Studio 规范
- XML：清晰缩进，复杂布局需注释
- Markdown：UTF-8 编码，使用 H1–H3 标题，可选 emoji
